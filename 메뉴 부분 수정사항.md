* **메뉴 카테고리**

  * **비밀번호 변경 카테고리** 계정 상태에 있는 비밀번호 변경은 정말 사용자가 생겨서 회원가입을 했을 때 누르게 되면 현재 비밀번호를 입력하시오라는 멘트가 나오면서 사용자의 비밀번호를 누를 수 있도록 하고, if 구문을 활용하여 True가 나온다면 변경할 비밀번호를 입력하시오라는 멘트가 나오면서 비밀번호를 변경 할 수 있도록 할 것.
  * **언어 설정 카테고리** 눌렀을 때 English로 바로 전환되도록 구성되어 있는데 DeepL API를 활용하여 영어 스페인어 아랍어 중국어 베트남어 인도네이사어를 언어설정을 눌렀을 때 카테고리에서 선택 할 수 있도록 하고, 만약 누르게 되면 정말로 언어 설정을 변경하시겠습니까? 라는 멘트가 나오고 밑에 예 아니오 라는 멘트가 적혀 있고 만약 예라는 멘트를 사용자가 클릭하게 디면 API가 번역을 하면서 동시에 웹사이트가 전체 재부팅이 되어 언어가 선택한 언어로 모두 번역되도록 설정할 것.
  * **다크모드 카테고리** 다크모드를 누르게 되면 시스템 기본값에서 다크모드라는 멘트가 달리면서 배경은 검정색 멘트들을 하얀색으로 바뀌도록 구현 할 것. 다른 것들이 다크모드가 되면서 보호색으로 잘 안보이게 되지 않도록 할 것.
  * **알림 설정** 알림 설정을 누르게 되면 켜기 끄기 카테고리가 나오고 켜기라는 버튼을 누르면 다른 사용자들이 공유하는 글들을 모두 알림을 통해 수신할 수 있도록 하고 끄기 버튼을 누르게 되면 다른 사용자가 보내는 모든 글들은 수신 차단 시키게 할 것.
  * **캐시 삭제** 캐시 삭제 카테고리는 그냥 완전 삭제 할 것.
  * **공지 사항** 이 웹사이트를 만든 내가 글을 올리게 되면 전체적으로 사람들이 공지사항 카테고리를 통해 내가 쓴 글들을 볼 수 있도록 연동시킬 것. 없으면 화면전환 후 중앙에 공지사항이 없습니다. 라는 멘트를 달 것.
  * **문의하기**: 문의하기를 누르게 되면 화면 전환이 되어 3개의 카테고리로 기능/시스템 관련 문의, 수식 추가 관련 문의, 일반 문의를 이름으로 만들어 주고 3개중 하나를 누르게 되면 또 다른 화면으로 전환되어 사용자들이 제목과 내용을 쓸 수 있는 창이 만들어 지도록 구현하고, 만약 다른 사용자들이 글을 써서 글을 올리게 되면 사이트 생성자인 나에게 알림이 오도록 구현할 것.
  * **로그아웃** 로그아웃 버튼을 누르게 되면 사용자가 회원가입을 통해 아이디 비번을 만들어 놓은 것들이 없어지고, 서브 로딩화면으로 이동하여 로딩하는 동작 없이 계산 생태계 제목 밑에 아이디 비번 입력창과 로그인 클릭창을 만들어서 다른 계정으로 로그인 할 수 있도록 함. 로그아웃을 하더라도 로그인을 통해 했는 활동들은 다시 로그인 했을 때에도 그대로 남아 있을 수 있도록 구현할 것.
  * **회원 탈퇴** 회원 탈퇴를 누르게 되면 정말로 회원을 탈퇴하시겠습니까? 라는 멘트와 함께 밑에 예 아니오 버튼을 누를 수 있도록 하고 예를 누를 경우 사용자의 비밀번호를 입력하시오라는 멘트와 함께 사용자의 비밀번호를 누를 수 있도록 하고 밑에 취소 확인 버튼을 누르어 취소를 누르면 다시 눌렀던 것들은 없어지고 다시 메뉴창을 돌아오도록 하고, 확인 버튼을 누르고 만약 틀릴시에도 똑같이 메뉴창으로 돌아오도록 하고, 만약 확인을 눌렀는데 일치 할 경우 사용자가 회원가입을 통해 했던 데이터들이 모두 삭제되고, 다시 회원가입을 통해 새로운 계정으로 로그인 할 수 있도록 서브화면에 아이디 비번창이 나오면서 회원가입을 할 수 있도록 하고, 웹사이트를 새로 들어온 사람도 회원가입 버튼을 통해 아이디 비번을 입력해서 들어갈 수 있도록 하고, **그냥 무조건 사이트에서 활동을 하려면 서브화면에 아이디 비번을 입력하거나 회원가입을 하고**, **서브화면에서 아이디 비번이 일치해 로그인에 성공할 경우 로딩을 2초 이내에 완료해 메인 화면으로 전환 할 수 있도록 할 것.**
