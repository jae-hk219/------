## 기술스택

* **프레임 워크:** react.js (최신 안정 버전)
* **개발 도구:** Vite (빠른 개발 환경 및 빌드 속도 확보)
* **스타일링:** Tailwind CSS (직관적인 유틸리티 클래스로 빠른 UI 구현 및 피그마 디자인 매칭)
* **아이콘 사용:** React Icons (피그마의 Iconify 아이콘들과 매칭)
* **상태 관리:** React Context API (가볍고 직관적인 상태 관리)
* **Routing:** React Router DOM (스플래시 화면에서 메인 화면으로의 전환 및 페이지 이동 처리)
* **AI API:** OpenAI API (GPT-4o mini)
* **계산 처리:** Axios (AI 서버와의 비동기 통신 및 수식 계산 요청 처리)

**게시글 목록 영역:**

* **디자인:** 게시물 목록 각각은 배경과 같은 #F5F5F5 이고, 배경과 같은 색임으로 구분이 될 수 있게 은은한 그림자와 부드러운 모서리로 구분 지어 주어야 함.
* **동작:** 게시물 목록이 있고, 목록을 눌렀을 때 게시물 내용을 볼 수 있도록 화면 전환 해야함.

  * 제목과 본문으로 구분지어 목록 내부에 저장하고, 제목의 내용을 그대로 옮겨 적되, 본문의 내용은 아무리 많아도 1줄 이내만 적어주고 나머지 내용은 '...'으로 대체
  * 게시물 목록 오른쪽 하단에는 Iconify에 좋아요 아이콘과 싫어요 아이콘을 사용해 작게 표시해 두고 각각의 아이콘 옆에는 숫자 0으로 시작하여 사람들이 게시글 내부에서 좋아요나 싫어요 버튼을 누르면 +1 되고 다시 한 번 누르면 -1이 되도록 함. (내부 게시글에서 좋아요와 싫어요를 누르는 건 4번 하단 탭 바 영역에서 설명.)

**게시글 내부 영역:**

* **구성:** 제목과 본문이 화면에 나오도록 하고, 화면 상단에는 앱 이름과 그 밑에 작게 "전기","배관","전체","공학"인지 앞 서 목록에서 정해진 내용에 따라 입력되어 질 것.

  또한 내용이 있고난 다음 하단 중앙 부분에 길이를 대칭을 가지고 직사각형 2개를 만들고, 그 안에 Like와 Dislike의

  아이콘을 Iconify를 활용하여 중앙에 넣고 밑에 작은 글씨로 앞 서 말했던 숫자를 넣어서 좋아요,싫어요를 한 번 누르면 +1 한 번 더 누르면 -1이 되고, 불연속 주기함수를 사용하여

  정수인 함수로 홀수 클릭은 +1 짝수 클릭은 -1이 되도록 함.
* **디자인:** 제목 부분은 Bold체로 굵게 하고 본문 내용은 Thin을 사용하여 제목과 본문을 구분짓는다.
