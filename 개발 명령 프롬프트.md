## 기술스택

* **프레임 워크:** react.js (최신 안정 버전)
* **개발 도구:** Vite (빠른 개발 환경 및 빌드 속도 확보)
* **스타일링:** Tailwind CSS (직관적인 유틸리티 클래스로 빠른 UI 구현 및 피그마 디자인 매칭)
* **아이콘 사용:** React Icons (피그마의 Iconify 아이콘들과 매칭)
* **상태 관리:** React Context API (가볍고 직관적인 상태 관리)
* **Routing:** React Router DOM (스플래시 화면에서 메인 화면으로의 전환 및 페이지 이동 처리)
* **AI API:** OpenAI API (GPT-4o mini)
* **계산 처리:** Axios (AI 서버와의 비동기 통신 및 수식 계산 요청 처리)

###서브 화면 (로딩 화면)

1. **개요**

* **화면 유형:** 모바일 앱 로딩 화면 (Splash Screen).
* **목표:** 앱 실행 시 표시되는 브랜드 로고, 제목, 부제목, 그리고 로딩 상태를 시각적으로 전달한다.
* **디자인 스타일:** 평평하고 그래픽적이며 전문적이고 신뢰할 수 있는 느낌.

**2. 전체 사양**

* **배경색:** 단색 짙은 남색 계열 (#1A1D29 계열의 차콜 네이비).
* **모든 텍스트 색상:** 흰색 (#FFFFFF).

**3. 중앙 로고 상세**

**중앙 로고 아이콘**

* **스타일:** 평평하고 그래픽적인 벡터 스타일.
* **구성:**

  * **렌치 (Wrench):** 짙은 회색 계열 (#4F5569 계열)의 조정 렌치 스패너 아이콘. 상단에 스패너 부분, 하단에 링 부분.
  * **번개 (Lightning Bolt):** 밝은 노란색 계열 (#FFEA00 계열)의 4점 번개 아이콘.
* **겹침 구조:** 노란색 번개 아이콘이 렌치 아이콘 위에 겹쳐서 중앙에 위치. (렌치 아이콘의 스패너 부분에 걸쳐짐).

**텍스트 영역**

* **정렬:** 모두 수직 중앙 정렬.

**제목 텍스트**

* **텍스트 내용:** "계산 생태계"
* **스타일:** 흰색 (#FFFFFF), 굵은(bold) 한국어 서체.
* **크기:** 화면 너비에 비해 상대적으로 크고 명확.

**부제목 텍스트**

* **텍스트 내용:** "전문가 현장 계산기 및 커뮤니티"
* **스타일:** 흰색 (#FFFFFF), 표준(regular) 또는 얇은 한국어 서체.
* **크기:** 제목 텍스트보다 작고 얇음.

**로딩 표시기**

* **스타일:** 흰색 (#FFFFFF)의 호(Arc) 모양의 진행률 스피너.
* **형태:** 반원 또는 약 3/4 원 모양의 깨끗한 선.
* **위치:** 화면 하단 중앙.
* **동작 (애니메이션):** 시계 방향으로 무한 회전하는 애니메이션 적용.

**레이아웃 명세**

* **모든 요소의 배치:** 화면 중앙에 수직으로 정렬된다
* **상대적 위치 (Top to Bottom):**

  1. 화면 상단에서 약 1/3 지점에 '중앙 로고 아이콘' 배치.
  2. 로고 아이콘 바로 아래에 '제목 텍스트' 배치.
  3. 제목 텍스트 바로 아래에 '부제목 텍스트' 배치.
  4. 화면 하단 중앙에 '로딩 표시기' 배치.

## 카테고리 및 계산기 데이터 명단

계산기 리스트는 아래의 고정된 구조를 가지며, 각 항목 클릭 시 매칭되는 수식 입력 폼 상태(State)로 전환되어야 합니다.

### 1. 전기 공사 (배지 색상: 주황/노란색 계열)

* **허용전류 및 전선 단면적 선정** (아이콘 텍스트: `I`)
* **전압강하 계산** (아이콘 텍스트: `ΔV`)
* **전력 계산 (단상/3상)** (아이콘 텍스트: `W`)

### 2. 배관 공사 (배지 색상: 푸른색/하늘색 계열)

* **배관 유량 및 유속 계산** (아이콘 텍스트: `Q`)
* **배관 압력 손실 계산** (아이콘 텍스트: `ΔP`)
* **펌프 양정 및 소요 동력 계산** (아이콘 텍스트: `HP`)

### 3. 일반 및 공학 계산 (배지 색상: 에메랄드/회색 계열)

* **단위 변환기 (길이/무게/압력 등)** (아이콘 텍스트: `U`)
* **기하학 계산 (도형 면적 및 입체 부피)** (아이콘 텍스트: `A`)
* **재료 중량 계산 (철판/파이프/콘크리트)** (아이콘 텍스트: `Kg`)
* **물리량 계산 (토크, 모멘트, 속도)** (아이콘 텍스트: `T`)
* **수학 기초 (분수, 백분율, 비례식)** (아이콘 텍스트: `%`)

###메인 화면(커뮤니티 화면)

* **목표:** 제공된 이미지를 바탕으로 'Everytime' 스타일의 모바일 커뮤니티 화면 UI를 개발하고자 합니다. 사용자가 커뮤니티 카테고리를 눌렀을 때 뜨는 화면을 구현해 주세요. 이미지에 나타난 균형 잡힌 배치와 정보 구조를 완벽하게 반영하는 것이 핵심입니다.

1. **상단 헤더 영역**

* **구성:** 좌측 뒤로 가기 아이콘, 중앙 카테고리 제목 및 부제목, 우측 검색 및 더 보기 아이콘 배치.
* **디자인:** 이미지와 같이 하단에 연한 회색 경계선(border-b)을 추가하여 본문과 구분해 주세요.

2. **카테고리 탭 영역**

* **구성:** "[전체]", "[전기공사]", "[배관공사]", "[일반공학]", "[질문/답변]" 탭으로 구성된 가로 스크롤 가능한 탭 바.
* **동작:** 사용자가 탭을 클릭하면 선택된 탭이 활성화 상태로 강조되고, 하단 게시글 목록이 필터링되는 상태(State) 로직을 구현해 주세요.

3. **게시글 목록 영역**

* **구성:** 스크롤 가능한 영역에 여러 개의 게시글 카드를 세로로 배치합니다.
* **데이터:** 이번 MVP 버전에서는 이미지의 내용을 제외하고, 각 카테고리별로 일반적인 플레이스홀더 데이터를 사용하여 3~4개의 게시글 카드를 구성해 주세요.

4. **게시글 상세 카드 구조**

각 게시글 카드는 균형 잡힌 배치를 따릅니다.

* **상단:** 게시글 제목 배치.
* **중앙:** 게시글 내용 요약 배치.
* **하단:** 작성 시간 및 작성자 정보를 좌측에, 댓글 수 아이콘 및 숫자를 우측에 배치해 주세요.
* **디자인:** 각 카드는 부드러운 모서리와 은은한 그림자(sm) 적용하여 배경과 구분해 주세요. 이미지 포함 카드 및 광고 카드와 같이 레이아웃이 변형되는 경우를 고려하여 구현해 주세요.

5. **플로팅 버튼**

* **구성:** 게시글 목록 영역 우측 하단에 원형 플로팅 버튼 배치.
* **디자인:** 아이콘 및 "글 쓰기" 텍스트로 구성되며, 이미지와 같이 배경색을 적용하여 강조해 주세요.

6. **하단 탭 바**

* **구성:** "홈", "커뮤니티", "계산기", "기록", "채팅" 5개의 메뉴 아이콘 가로 배치.
* **디자인:** 상단에 연한 그림자(shadow-[0_-4px_10px_rgba(0,0,0,0.05))를 추가하여 본문과 구분하고, 현재 활성화된 "커뮤니티" 메뉴를 시각적으로 강조해 주세요.

**코드 확장성 및 요구사항**

* **컴포넌트 구조:** 유지보수가 용이하도록 PostCard, CategoryTab, FloatingActionButton, BottomNav 등으로 컴포넌트를 분리하여 작성해 주세요.
* **데이터 배열 기반:** 게시글 목록과 카테고리 탭 목록을 컴포넌트 내부에 하드코딩하지 말고, map() 함수를 사용하여 데이터 배열을 기반으로 동적으로 렌더링되도록 설계해 주세요.
* **반응형 디자인:** 모바일 퍼스트 반응형 디자인을 적용하여 다양한 화면 비율에서도 UI가 깨지지 않도록 가로 폭 최대치(max-w-md)를 지정하고 중앙 정렬해 주세요.
* **스타일링:** Tailwind CSS를 사용하여 각 요구사항을 직관적으로 구현해 주세요 (예:border-b, shadow-sm, rounded-xl, fixed, max-w-md mx-auto).

###메인화면(메뉴 화면)

**개요**

제공된 이미지들의 레이아웃 구조를 참고하여, 다크 모드 기반의 깔끔하고 직관적인 '내 정보' 설정 화면을 개발해 주세요. 복잡한 기능은 제외하고 필요한 핵심 메뉴와 '언어 설정' 기능을 추가하는 것이 목적입니다.

**기술 스택 및 스타일 가이드**

* **Framework:** React.js (Vite 환경)
* **Styling:** Tailwind CSS 기반의 밝은 흰색 모드 테마
* **Text Color:** 주 텍스트는 검정(#000000), 부가 설명 및 화살표는 진한 회색

**UI/UX 구성 요소 및 배치의 세부 요구사항**

* **상단바:** 좌측에 뒤로가기 아이콘(`<`), 중앙에 "내 정보" 타이틀 배치. 하단에 구분선 추가.

**통합 프로필 카드**

* **형태:** 가로로 긴 단일 직사각형 카드 형태 (`bg-zinc-800` 또는 내부 음영 적용).
* **내부 배치:**

  * 좌측: 원형 프로필 기본 아바타 아이콘.
  * 중앙 우측: 상단에 **사용자 이름**, 하단에 **전공 및 활성화된 게시판 정보**를 작은 글씨로 배치 (예: 홍길동 / 컴퓨터공학부 · 전기공사).
  * 우측 끝: 프로필 수정을 암시하는 오른쪽 화살표(`>`) 아이콘 배치.

**설정 메뉴 그룹**

각 메뉴 그룹 위에는 카테고리 이름(소제목)이 붙고, 항목들은 세로 리스트로 나열됩니다. 항목 우측 끝에는 화살표(`>`)나 현재 상태 값이 표시됩니다.

* **[계정 설정] 그룹**
  * **비밀번호 변경**
  * **언어 설정:** 우측 끝에 현재 선택된 언어인 '한국어' 텍스트를 회색으로 표시. (클릭 시 언어를 변경할 수 있는 모달이나 드롭다운의 상태 로직을 준비해 주세요.)
* **[앱 설정] 그룹**
  * **다크 모드:** 우측 끝에 '시스템 기본값' 표시.
  * **알림 설정**
  * **캐시 삭제:** 클릭 시 '캐시가 삭제되었습니다'라는 알림 팝업이 뜨는 연동.
* **[이용 안내] 그룹**
  * **공지사항**
  * **문의하기**
  * **로그아웃 / 회원 탈퇴:** 이 두 메뉴는 리스트 최하단에 약간의 여백을 두고 붉은색 톤이나 흐린 회색 톤으로 독립 배치하여 실수로 누르지 않게 구분.

1. **데이터 기반 렌더링:** 메뉴 아이템들은 하드코딩하지 말고, `sections = [{ title: '계정 설정', items: [...] }]` 형태의 배열 구조로 만들어 `map` 함수로 깔끔하게 출력해 주세요.
2. **언어 토글 상태(State):** '언어 설정' 항목을 클릭했을 때 한국어/영어 간의 상태 변경이 이루어지는 `currentLanguage` useState 로직의 뼈대를 만들어 주세요.
3. **컴포넌트 분리:** 전체 페이지 컴포넌트 내에서 `ProfileCard`와 `SettingItem` 컴포넌트를 분리하여 가독성을 높여주세요.
